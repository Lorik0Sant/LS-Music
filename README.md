# LS Music 🎵💿

Десктоп-приложение для стримеров: зритель тратит **баллы канала Twitch**, пишет
название трека — и трек играет в OBS с **анимацией виниловой пластинки**, которую
можно включать и выключать прямо из приложения.

Источники музыки — **Яндекс.Музыка** и **Spotify**. Работает и **без премиума**
(у кого что есть): бесплатные аккаунты слышат рекламу — точно так же, как в самих
приложениях.

```
Зритель тратит баллы канала
        │
        ▼
  LS Music слушает Twitch EventSub (WebSocket)
        │
        ├─► ищет трек в Яндекс.Музыке по названию
        ├─► добавляет в очередь
        └─► играет в OBS Browser Source + крутит винил
```

## Возможности

- 🎧 Заказ треков за Channel Points (Twitch EventSub, без публичного сервера).
- 💿 Винил-оверлей для OBS с плавными анимациями — **тумблер вкл/выкл** в один клик.
- 📋 Очередь заказов: «сейчас играет», пропуск, очистка, ручной тест-заказ.
- 🔊 Регулировка громкости и переключатель карточки «сейчас играет».
- 🧩 Провайдеры музыки через единый интерфейс — Spotify добавляется без переделки.
- 📝 Живой лог событий.

## Технологии

Electron + TypeScript + Vite (electron-vite), React (панель управления),
Express + ws (локальный сервер оверлея).

## Установка для разработки

Нужен **Node.js 18+**.

```bash
npm install      # установка зависимостей
npm run dev      # запуск в режиме разработки
npm run build    # сборка
npm run dist:win # собрать «LS Music.exe» (портативный) в папку release/
```

> `dist:win` делает **портативный** `LS Music.exe` — один файл, запускается без
> установки. Нужен установщик NSIS? Поменяйте target на `nsis` в
> `electron-builder.yml`.

> Если Electron ругается `failed to install correctly`, бинарник не докачался —
> повторите `npm install` или запустите `node node_modules/electron/install.js`.

## Настройка (3 шага)

### 1. Twitch — войти по кнопке

Channel Points доступны для аккаунтов уровня **Affiliate/Partner**.

Нажмите **«Войти через Twitch»** — откроется окно входа, логинитесь, разрешаете
права, готово. Один раз нужно указать Client ID:

1. <https://dev.twitch.tv/console/apps> → **Register Your Application**.
2. Name — любое; **OAuth Redirect URLs** — `http://localhost`; Category — *Application Integration*.
3. Скопируйте **Client ID** в раздел «Расширенно» у Twitch (или зашейте в код:
   `DEFAULT_TWITCH_CLIENT_ID` в [`src/main/oauth.ts`](src/main/oauth.ts)).

Затем **«Загрузить награды»** и выберите награду Channel Points, по которой
заказывают треки (или оставьте «любая награда»). Зритель пишет название трека
**в поле ввода награды** при активации.

### 2. Источник музыки (Яндекс и/или Spotify)

Выберите активный сервис в блоке **«Источник музыки»**. Токены вручную вводить
не нужно — просто вход в аккаунт. У каждого сервиса — два режима воспроизведения.

**Яндекс.Музыка** — нажмите **«Войти через Яндекс»**, залогиньтесь в открывшемся
окне, разрешите доступ. Токен подтянется автоматически. Режимы:
- *Поток (Плюс)* — полный трек играет прямо в оверлее (нужен Яндекс Плюс).
- *В приложении (без Плюса)* — трек открывается в приложении/вебе Яндекс.Музыки
  (бесплатно, с рекламой). Оверлей показывает винил, звук идёт из приложения.

**Spotify** — нажмите **«Войти через Spotify»** (вход через окно, PKCE, без
client-secret). Режимы:
- *Приложение (полный трек)* — трек запускается в установленном клиенте Spotify.
  **Без Premium играет с рекламой**, с Premium — без.
  *(нужен установленный десктоп-клиент Spotify)*
- *Превью 30 сек* — фрагмент играет прямо в оверлее, без клиента и рекламы
  (если у трека есть превью).

> **Один раз для Spotify**: нужен свой Client ID. Создайте приложение на
> <https://developer.spotify.com/dashboard> → *Create app*, добавьте Redirect URI
> `http://127.0.0.1:8765/callback`, вставьте **Client ID** в раздел «Расширенно».
> Можно зашить его в код: `DEFAULT_SPOTIFY_CLIENT_ID` в
> [`src/main/oauth.ts`](src/main/oauth.ts).

> Токены хранятся локально в `settings.json` и отправляются только в официальные
> API Яндекс.Музыки и Spotify.

### 3. OBS

1. Скопируйте адрес оверлея из приложения (по умолчанию `http://127.0.0.1:7895/`).
2. В OBS добавьте источник **Browser** (Браузер) с этим URL.
3. Размер — под ваш экран (например 1920×1080), фон оверлея прозрачный.
4. Браузер в OBS разрешает автозапуск звука — трек заиграет сам.

Тумблер **«Анимация винила вкл/выкл»** меняет оверлей мгновенно, без
перезагрузки источника.

## Где хранятся настройки

`%APPDATA%/LS Music/settings.json` (Windows). Токены — только там, локально.

## Как устроено воспроизведение

Каждый трек играется одним из двух способов (выбирается режимом провайдера):

- **audio** — оверлей сам проигрывает прямой звук (`<audio>`): поток Яндекс Плюс
  или 30-сек превью Spotify. Очередь двигается по событию окончания трека.
- **external** — трек отдаётся официальному приложению (`spotify:track:…` или
  страница Яндекс.Музыки). Бесплатные аккаунты слышат рекламу. Оверлей показывает
  только винил, а очередь двигается по таймеру длительности трека.

Почему Spotify не играет напрямую: Web API Spotify **никогда** не отдаёт
аудиопоток, а управление плеером требует Premium. Поэтому универсальный путь,
работающий и без Premium, — запуск трека в установленном клиенте Spotify.

## Структура

```
src/
  main/                 # Electron main (Node)
    index.ts            # запуск окна и сервисов
    config.ts           # хранилище настроек
    overlay-server.ts   # Express + WebSocket для OBS
    queue.ts            # очередь воспроизведения
    status.ts, bus.ts   # статус и шина событий
    twitch/             # auth (Device Code Flow) + EventSub
    music/              # provider, yandex, spotify, registry
  preload/index.ts      # contextBridge API
  renderer/             # панель управления (React)
overlay/                # статический оверлей для OBS (винил + аудио)
```
